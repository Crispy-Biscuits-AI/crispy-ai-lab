# Security

`crispy-ai-lab` is an experimental local runtime stack for CrispyBrain. It is meant for self-hosted local development and demo work, not unattended production deployment.

## Please Do Not Commit

- `.env` files
- passwords, tokens, or API keys
- private workflow exports with sensitive data
- private memory content or customer-derived examples
- machine-specific paths unless they are intentionally generalized for docs

## Reporting

If you find a vulnerability or accidentally discover a secret in tracked files, do not post the raw secret publicly.

Use a private maintainer channel if one is available. If GitHub is the only path, open a redacted report that explains the issue without exposing sensitive values.
